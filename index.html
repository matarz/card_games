Card Games

A proof of concept for using OOP in c++

Download as .zip Download as .tar.gz View on GitHub

This is a proof of concept for using Object Oriented Programming in C++. The assignment was to build a program that would provide general functionality for building one or more types of games (card games, boardgames, dice games, etc) and then use it to build a game as an example. I went with card games for a type and built a simple Uno game. Here is a snippet of the assignment:

For this program, I want you to design an Object Oriented Program using C++ that will allow an application programmer to easily implement one of the above games in software. So, although you should test this out with one of the games mentioned (or others)…the real goal is a more generalized set of software that would be useful to support many different games. Think of common operations that would be needed regardless of the game. Of course the media is important – is it a board game, dice, cards, words? This needs to be taken into account with your program.

The full assignment can be found here.

When looking at the code, you would notice that general functionality include things that apply to most card games such as shuffling and dealing cards. Other functionality and game rules that are specific for a game are built into other classes.

There is also a design document that describes how classes interact with each other and what they do.
